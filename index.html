overview

Now that I've learned how to create a cohesive visual for websites and how to use hierarchical elements to demonstrate importance and guide the flow of a website, it was time to learn how to make it actually functional and interactive!

Thus, with this site, I explored the process of using multiple interface components and tying those components to an internal data structure to develop an interface that is fully functional to serve customers who are growing for potential boba options.


why (is this important)?

While one half of UI/UX is design, the other half is user interaction. And having a convenient system in which users will not be confused using is a huge aspect in what makes or breaks a website. A large part of the user interaction is figuring out the target audience of your site and try to tailor the components of the interactions to their needs. Thus…

steps

since my boba store will mainly be targeting those who will use the site to browse for their favorite items or to scrolling through to discover new items to try, I implemented features that would best suit their needs and would allow them to get to their wanted item quickly.

step one:
sort & filter


I first implemented a sort section that would allow the user to sort the items by default (sort by popularity of drinks) or to sort by prices low to high.

I also implemented two filters that let the users shop/browse the items they want to order by category (in this case: Milk Tea vs Specialty and Caffeinated vs Decaffeinated).

see interaction of components here:

step two:
add to cart (aggregator)


An ordering site surely cannot be complete without a cart component! Thus, I also created an aggregator component which is displayed on the side of the interface so that users can see immediately what items were added and what can be removed. In addition:

• The menu item contains both an add to cart and remove from cart button as opposed to one single button that changes states (from “add” to “remove”) when clicked so that users can add/remove an item multiple times.

• And when the cart is empty, it gently nudges the user to start shopping!

Additional Note!

The Aggregator component extends from App ( <Aggregator cart={cart}> </Aggregator > in App) and receives its menu items data from menu-data.json. The MenuItem component from extends from App and receives its menu items data from menu-data.json. When the aggregator data is updated, the props change and the aggregator renders with the updated data.

step three:
creating the final website using react (yay!!)


finally, using both the Aggregator and MenuItem components to handle the passed in item information, I constructed the site such that the state and statues of the items are changed/State Change is triggered when we click on a filter type, a sort by type, or add to cart, resulting in the corresponding filtering/sorting/adding/removing actions. check it out here!

reflection

From working on the interactive elements on this site, I was able to get a better idea of how components worked individually as well as in relation to each other. I also got a deeper understanding of how State Change works to update the data passed down in the lists. Although I am mostly happy with how the site looks, in the future, I would try to align the filter checkboxes better so that they do not have different margins. I would also try to make “add to cart” a separate pop up that opens up when you click on it.
